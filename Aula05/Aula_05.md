# Aula [05] - [Introdução ao Scratch]

## 💡 O Conceito
Nesta aula, o foco foi entender a base lógica dos algoritmos utilizando uma linguagem visual de blocos para facilitar a visualização do fluxo.

### 📚 Principais fundamentos:
- **Algoritmos:** É o caminho para a solução de um problema. Uma sequência finita de passos para atingir um objetivo.
- **Scratch:** Uma linguagem de programação visual baseada em blocos, ideal para treinar a lógica sem se preocupar com erros de digitação ou sintaxe.
- **Pense de Forma Lógica:** Antes de escrever o código, você precisa entender a sequência: Início -> Processamento -> Fim.

---
## 🏗️ Estrutura Mental (O "Mapa" da Aula)

Aqui está como eu organizei o raciocínio dessa parte teórica:

```text
Algoritmos no Scratch

 ├── Eventos (Gatilhos)
 │    └── Ex: Quando a bandeira verde for clicada
 ├── Movimento (Ações)
 │    └── Ex: Mova 10 passos / Gire 15 graus
 ├── Aparência (Interação)
 │    └── Ex: Diga "Olá!" por 2 segundos
 ├── Controle (Fluxo)
 │    └── Ex: Espere 1 segundo / Repita 10 vezes
 └── Sensores (Condições)
      └── Ex: Tocando em "cor tal?" ou "ponteiro do mouse?"

```

---

## 📝 Anotações de Suporte
**Destaque**: O Scratch ajuda a entender que a ordem dos blocos altera completamente o resultado final. Um passo no lugar errado quebra a lógica.

**Observação**: No Scratch, os personagens são chamados de "Atores" e os cenários de "Palcos". Cada ator pode ter seu próprio script (algoritmo).



## 🚀 Insight do Dia
**Dica de Ouro**: Não subestime os blocos. A lógica que você constrói encaixando peças no Scratch é a mesma que você vai usar escrevendo linhas de código no VisualG e no JavaScript.
